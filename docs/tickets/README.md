## ステータス定義
- **Ready**: 着手可能
- **In Progress**: 作業中
- **Review**: レビュー待ち
- **Done**: 完了

## 優先度
- **High**: MVP必須
- **Medium**: MVP推奨
- **Low**: MVP後でも可

## 並行開発のガイドライン

### 独立して作業可能なチケット
以下のチケットは他のチケットに依存せず、並行で作業可能です：
- PA-001（データベースセットアップ）
- PA-004（フロントエンドUI改善）

### 依存関係
- PA-002, PA-003, PA-005, PA-006, PA-008 → PA-001 が前提
- PA-007 → PA-004 が前提

## 作業の進め方
1. チケットファイルを確認
2. 作業開始時にステータスを「In Progress」に更新
3. 完了したらステータスを「Done」に更新
4. PRを作成する場合はチケットIDをコミットメッセージに含める
